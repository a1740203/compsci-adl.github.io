Mallard Makers

Game Programming Competition

Overview

Mallard Makers is a programming competition, where teams get together to design and develop their own game, and be in the chance to win some sweet prizes. It is a chance to finally fulfill that itch to work on your own project, and create something you want to do.

This quite an open ended competition, but there will be an overarching theme that should be taken into consideration. You do not need to hand in any design documents. Your code will not be scrutinized for marks. In the end, it's the final product in all its glory (and bugs) that will be looked at.

Judging Each entry will be judged by a small panel of judges. Each submissions should include a short video that demonstrates all the features of your game. This will be the main point of judging for all minor awards and the People's Choice Award. Any major awards, will be run, played and tested. Source code should be provided for all submissions, but commenting/style/format will not be considered.
Criteria The submissions will be based around the following criteria:
  • Design concept
  • Core mechanics
  • Aesthetics
  • Theme involvement
Prizes Major Prizes
  • Golden Egg. The best game form out of all the teams.
  • Golden Egg: Junior. The best game from a junior team.
  • People's Choice. Publicly voted best game out of all games.

Rules

Language and Libraries
  • Can use any free programming language.
  • Can use any library or framework (must be included in source).
  • Can NOT use pre-existing engine (eg: Unity, GameMaker, Unreal Engine).
  • Strive to be able to run on University systems, both performance wise and technical wise.
Teams and Participants
  • Teams can be of up to 3 people.
  • Participants do NOT have to be a computer science students.
  • Participants must be current students at time of competition start.
  • Participants do not have to be from the same year level.
  • A team can only qualify as a "Junior Team", if all of the participants are in their first year of computer science (or are not computer science students).
  • The majority of the team must be from the University of Adelaide. Any other participants still must be students.
Submission
  • Source code must be available for review and judging.
  • An accompanying video, demonstrating your game and all its major features is required. Upload the video to some streaming service (eg: YouTube.com, Vimeo.com), and then provide the link.
  • A text README is also required, that includes the following:
    • Instructions on how to build and run the game.
    • Dependencies required to build the game.
    • Game controls and instructions.
  • Submission of code is done by any of the following (most preferred to least prefered method):
    • Web link submission (eg: github.com)
    • Email to competition@csclub.org.au
    • Physical handin to committee member (Sam)
